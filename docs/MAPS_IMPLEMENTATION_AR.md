# تنفيذ تحسينات الخرائط في تطبيق جولة للسائق

## ما يعمل داخل هذا المستودع

1. **MapLibre متجهي نهاري:** الشاشات الرئيسية والرحلة وبين المحافظات تستخدم
   نمط MapLibre واحدًا مع حدود العراق من zoom 6 إلى 18.
2. **فتح بلا مربعات:** يبقى غطاء خفيف موحد حتى يصل حدث MapLibre Idle، أي بعد
   تحميل البلاطات المطلوبة وانتهاء الانتقالات. توجد مهلة 12 ثانية وزر إعادة
   محاولة، ويسجل التطبيق زمن أول إطار مكتمل بالمللي ثانية.
3. **GPS Map Matching:** يحتفظ التطبيق بآخر ثماني قراءات GPS ويرسل الإحداثيات
   والدقة والتوقيت إلى OSRM Match. تظهر سيارة السائق وتُرسل إحداثيات العمل
   من النقطة المطابقة، مع رجوع آمن إلى GPS الخام عند ضعف الثقة أو تعطل OSRM.
4. **إعادة توجيه ذكية:** المسار محفوظ ولا يعاد حسابه عند كل حركة. يلزم ثلاث
   قراءات متحركة متتالية خارج ممر لا يقل عن 45 مترًا، ويتسع الممر مع دقة
   GPS، مع مهلة 20 ثانية بين عمليتي إعادة توجيه.
5. **نقطة التقاط ذكية:** اختيار نقطة على خريطة بين المحافظات يفحص OSRM
   Nearest. إذا كانت نقطة الطريق الأنسب مختلفة بوضوح، يرى المستخدم اقتراحًا
   ويختار قبوله أو إبقاء اختياره الأصلي.
6. **ETA محسّن:** يبدأ من مدة OSRM المبنية على نوع الطريق وسرعته، ثم يضيف
   معامل ساعة الذروة/اليوم ويمزج السرعة الحالية ضمن حدود تمنع القفزات، ويولد
   مجال ثقة أدنى وأعلى.
7. **بحث محلي:** لا يوجد أي استدعاء إلى Nominatim العام. عنوان الخدمة
   `GEOCODING_BASE_URL` ويُحصر البحث داخل العراق مع تفضيل العربية.
8. **خرائط دون اتصال:** من الإعدادات يمكن تنزيل محافظة، رؤية التقدم والحجم،
   تحديثها أو حذفها، مع خيار Wi-Fi فقط. التنزيل يستخدم مخزن MapLibre الأصلي.
9. **مناطق ذكية:** التطبيق يقرأ FeatureCollection كل ثلاث دقائق ويعرض مضلعات
   هادئة حسب مستوى الطلب. لا يستقبل التطبيق مواقع ركاب منفردة.
10. **تحديث بيانات دون انقطاع:** TileServer GL وOSRM يعملان بنظام
    blue/green خلف Nginx. لا يتحول المؤشر إلا بعد بناء الإصدار وفحصه.

## عقود Backend المطلوبة

لا يحتوي هذا المستودع على كود Backend أو تطبيق الراكب، لذلك أُضيف العقد
من جانب تطبيق السائق ولم تُخترع استجابات وهمية.

### GET /api/v1/maps/smart-zones

يرجع GeoJSON FeatureCollection. كل Feature من نوع Polygon ويحتوي فقط:

```json
{
  "id": "baghdad-karrada-20260723T1945Z",
  "kind": "demand",
  "demand": "high",
  "validUntil": "2026-07-23T19:50:00Z"
}
```

القيم المقبولة للطلب: `moderate`، `high`، `very_high`. على Backend جمع
الطلبات داخل خلايا أو مضلعات وتطبيق حد أدنى للعدد، ومنع إرسال إحداثيات راكب
أو معرّفه. مدة التحديث المقترحة ثلاث دقائق.

أنواع `kind` المدعومة: `official_pickup` و`airport_pickup` و`university_gate`
و`hospital_gate` و`complex_gate` و`no_pickup` و`no_stopping` و`danger`
و`closure` و`demand` و`service_area` و`city_boundary` و`pricing` و`promotion`.

## التحقق والمقاييس

- `flutter analyze`: بلا مشكلات.
- `flutter test`: 91 اختبارًا ناجحًا، بينها OSRM Nearest/Match، سياسة
  الانحراف، وETA.
- Android debug APK: بُني بنجاح مع MapLibre الأصلي.
- iOS debug Runner: بُني بنجاح دون توقيع مع MapLibre الأصلي.
- فحص الخادم المحلي: style وsprite وglyph وvector tile كلها HTTP 200.
- اختبار OSRM فعلي في بغداد: Route نجح، وMatch نجح بخمس نقاط وثقة 0.979.
- البحث المحلي الفعلي: بحث «بغداد» والبحث العكسي لإحداثيات وسط بغداد نجحا
  وأعادا أسماء وعناوين عربية من قاعدة العراق المحلية.
- على الجهاز المحلي بعد إحماء الخدمات: ملف التصميم 15–24ms، بلاطة متجهة
  نموذجية 4–14ms، وطلب Route ‏8–30ms، والبحث العربي النموذجي نحو 196ms.
  هذه أزمنة الخادم وليست زمن الرسم على الهاتف.

لا توجد قراءة موثوقة لزمن الإطار الأول قبل التغيير لأن النسخة القديمة لم
تكن تسجل هذا القياس. النسخة الجديدة تسجل القياس بعد اكتمال `MapEventIdle`
ولا تعرض الخريطة قبله؛ لذلك يجب جمع خط أساس «بعد» على جهاز Android وiPhone
فعليين وشبكة محددة قبل إعلان رقم نهائي أو مقارنة غير حقيقية.

للاختبار الميداني على أجهزة ضعيفة، اجمع:

- زمن `Jowla vector map first complete frame`؛ الهدف أقل من ثانيتين على
  الشبكة المحلية وأقل من أربع ثوانٍ على شبكة هاتف جيدة.
- نسبة عمليات Match المقبولة، متوسط مسافة snap، وعدد إعادة التوجيه لكل رحلة.
- خطأ ETA المطلق عند الوصول، ويُقسم حسب المحافظة والساعة دون تخزين مسار
  المستخدم الخام في التحليلات.
- معدل أخطاء style/tile/glyph وحجم الخرائط المنزلة.

## تطبيق الراكب

ينبغي نقل الملفات المشتركة الخاصة بـMapLibre وOSRM Nearest والبحث المحلي
والخرائط دون اتصال إلى حزمة Flutter مشتركة عند إتاحة مستودع تطبيق الراكب.
منطق اختيار الاقتراح أو إبقاء النقطة الأصلية جاهز في تطبيق السائق ويمكن
استخدام العقد نفسه دون تكرار.

## أهم الملفات المضافة أو المعدلة

- `lib/core/maps/`: إعداد حدود العراق ومكوّن MapLibre المتجهي وحالة التشغيل.
- `lib/core/services/road_route_service.dart`: Route وAlternatives وNearest
  وMatch وETA.
- `lib/core/services/road_position_matcher.dart`: مطابقة أثر GPS والرجوع
  الآمن إلى الإحداثيات الخام.
- `lib/core/services/smart_route_controller.dart`: كشف الانحراف وإعادة
  التوجيه مع المهلة والاحتفاظ بالمسار السابق.
- `lib/core/services/place_name_service.dart`: البحث والعكس عبر الخدمة
  المحلية فقط، مع انحياز للموقع وترتيب بالمسافة/الأهمية وذاكرة مؤقتة.
- `lib/core/services/smart_zones_service.dart`: عقد المناطق الذكية.
- `lib/features/offline_maps/`: تنزيل المحافظات وتحديثها وحذفها.
- `lib/features/home/` و`lib/features/trip/` و`lib/features/intercity/`:
  دمج الخريطة والمطابقة والتوجيه ونقطة الالتقاط والطبقات.
- `maps/`: TileServer GL وOSRM وNominatim وبوابة blue/green وسكربتات
  التجهيز والتحديث الذري.
- `test/road_route_service_test.dart` و`test/smart_route_controller_test.dart`:
  اختبارات منطق الخدمات الجديدة.

## القيود المتبقية

- لا يحتوي المستودع على Backend؛ لذلك `/maps/smart-zones` عقد جاهز في
  التطبيق وليس خدمة منشورة بعد.
- مرحلة ETA الثانية تقبل `historicalFactor` وتملك رجوعًا دائمًا إلى OSRM،
  لكن لا توجد في Backend الحالي خدمة إحصاءات رحلات مجمعة لإنتاج هذا المعامل؛
  المعروض حاليًا يستخدم OSRM والوقت/الذروة والسرعة الفعلية مع مجال ثقة.
- تقدير حجم المحافظة قبل التنزيل تقريبي ومحافظ؛ يظهر الحجم الحقيقي أثناء
  التنزيل وبعده. يلزم اختبار تنزيل فعلي على Android وiPhone للتأكد من سياسة
  النظام ومساحة القرص لكل جهاز.
- جودة الاختلافات الإملائية والمعالم المحلية تتبع بيانات OSM المستوردة.
  إضافة قاموس أسماء عراقية خاص تحتاج ملف بيانات مُراجع ودورة تحرير مستقلة.
- مستودع تطبيق الراكب غير متاح، لذلك لم تُنسخ التغييرات إليه. الواجهات
  والعقود المشتركة جاهزة للنقل عند توفيره.
